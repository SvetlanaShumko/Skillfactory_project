#Регистрация

***Цель***: регистрация нового пользователя
***Акторы***: пользователь (не зарегистрированный), система
***Контекст***: регистрация нового пользователя до/после выбора пиццы
***Предусловия***: пользователь не зарегистрирован в системе

##Основной сценарий:
1. Пользователь вводит номер телефона для регистрации
2. Система генерирует код доступа
3. Система отправляет смс с кодом доступа
4. Пользователь получает код доступа в смс
5. Пользователь вводит код доступа на сайте
6. Система регистрирует пользователя

##Исключения основного сценария:
3. Система отправляет смс с кодом доступа
4. Пользователь не получает смс в течение N минут
5. Система на сайте отображает сообщение «проверьте указанный номер телефона и нажмите кнопку «отправить код повторно»
6. Пользователь проверяет введенный номер телефона и нажимает кнопку «отправить код повторно»
7. Система генерирует код доступа
8. Система отправляет смс с кодом доступа
9. Пользователь получает код доступа в смс
10. Пользователь вводит код доступа на сайте
11. Система регистрирует пользователя

##Альтернативный сценарий:
1. Пользователь вводит e-mail
2. Система генерирует пароль
3. Система отправляет письмо с паролем
4. Пользователь получает письмо с паролем
5. Пользователь вводит пароль на сайте
6. Система регистрирует пользователя

##Исключение альтернативного сценария:
3. Система отправляет письмо с паролем
4. Пользователь не получает письмо в течение N минут
5. Пользователь нажимает на сайте кнопку «отправить пароль повторно»
6. Система отображает окно с введенным пользователем e-mail и сообщением на окне «проверьте e-mail»
7. Пользователь нажимает кнопку на окне «отправить пароль повторно»
8. Система генерирует пароль
9. Система отправляет письмо с паролем
10. Пользователь получает письмо с паролем
11. Пользователь вводит пароль на сайте
12. Строка изменена первый раз
